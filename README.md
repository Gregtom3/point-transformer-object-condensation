# Point Transformer V3 + Object Condensation for Particle Reconstruction

## Project overview

This repository is a scaffold for a **multi-subdetector particle
reconstruction pipeline**. The physical setting is a collider event
that leaves hits in multiple subsystems — silicon tracker, ECAL, HCAL,
RICH, etc. — each of which records a different native representation of
the passing particle. The goal of reconstruction is to take the full
union of hits and recover the set of particles that produced them,
together with each particle's energy, momentum, and PID.

We treat each event as an unordered, variable-size point cloud whose
points live in 3D detector space. Hits from different subdetectors are
first embedded into a common feature width with per-subdetector linear
encoders and tagged with a learned subdetector-type token. The unified
token stream is fed through a **Point Transformer V3** (PTv3) backbone,
which provides a permutation-invariant, serialization-based attention
mechanism that scales well to O(10^4-10^5) hit events.

On top of the per-hit PTv3 features we attach three **Object
Condensation** (OC) heads: a condensation likelihood `beta`, a cluster
coordinate `x` in a learned low-dim space, and a payload head that
predicts the per-particle quantities (energy, momentum, PID). At
inference, hits above a beta threshold act as condensation points and
claim nearby hits in `x`-space, turning one-stage grid-free prediction
into a concrete particle list.

## Architecture at a glance

```
per-hit raw features
   │
   │  per-subdetector nn.Linear + learned subdetector-type embedding
   ▼
unified (N, D) token stream
   │  + 3D coord, + offset (batch layout)
   ▼
PointTransformerV3 backbone (serialized attention, enc/dec U-Net)
   │
   ▼
per-hit features (N, D_out)
   │
   ├── beta head  (sigmoid)          → condensation likelihood
   ├── x head                        → cluster coordinates
   └── payload head (E, p, PID)      → per-hit per-particle prediction

Inference: β > t_β ⟹ condensation point. Claim hits within ||x - x_c|| < t_d.
```

## Third-party components

Both backbone and loss code are vendored as **git submodules** under
`third_party/`:

| Submodule | Upstream | Pinned SHA | License |
|---|---|---|---|
| PointTransformerV3 | <https://github.com/Pointcept/PointTransformerV3> | `3229e9b7de1770c8ad17c316f8e349982de509f8` | MIT |
| object_condensation | <https://github.com/object-condensation/object_condensation> | `926827d7cf743a946a6738f8f590f6ad07644b6f` | MIT |

`PointTransformerV3` is imported directly from its source tree — it is
not pip-installable. The wrapper at `src/models/backbone.py` prepends
its path to `sys.path`. `object_condensation` is a real Python package
and is installed in editable mode via
`pip install -e third_party/object_condensation[pytorch]` (see
`setup.sh`).

## Installation

The CUDA-linked wheels (`torch`, `torch-scatter`, `torch-geometric`,
`spconv`, `flash-attn`) must be installed in a particular order from
matching wheel indices. The easiest path is:

```bash
# clone with submodules
git clone --recursive https://github.com/gregtom3/point-transformer-object-condensation.git
cd point-transformer-object-condensation

# bootstrap (defaults to CUDA 12.1; override with CUDA=cu118 for 11.8)
CUDA=cu121 bash setup.sh
```

Manual pip path (CUDA 12.1 shown; swap `cu121` → `cu118` to match your
driver):

```bash
pip install --extra-index-url https://download.pytorch.org/whl/cu121 "torch>=2.1"
pip install --find-links https://data.pyg.org/whl/torch-2.1.0+cu121.html torch-scatter torch-geometric
pip install spconv-cu121
pip install flash-attn --no-build-isolation      # optional, see below
pip install numpy scipy pandas pyyaml omegaconf hydra-core timm addict \
            wandb tensorboard pytest
pip install -e third_party/object_condensation[pytorch]
```

Conda path:

```bash
conda env create -f environment.yml
conda activate ptv3-oc-reco
```

### About flash-attn

PTv3 prefers [FlashAttention](https://github.com/Dao-AILab/flash-attention),
which requires CUDA ≥ 11.6 and a reasonably modern Ampere+ GPU. If you
can't install it (CPU-only dev box, older GPU, build pain), **disable it
and shrink the patch sizes** in the model config:

```yaml
# configs/model/ptv3_base.yaml (or train/default.yaml)
backbone:
  enable_flash: false
  enc_patch_size: [128, 128, 128, 128, 128]
  dec_patch_size: [128, 128, 128, 128]
```

## Quickstart

```bash
python scripts/train.py --config configs/train/default.yaml
```

The default config points at a tiny dummy dataset so the pipeline
exercises end-to-end without real data. Swap in a real `HitDataset`
implementation once your data is ready.

To sanity-check the install:

```bash
pytest tests/
```

## Submodule maintenance

After a fresh clone without `--recursive`, populate the submodules
with:

```bash
git submodule update --init --recursive
```

To pull in newer upstream commits (explicitly — the submodules are
pinned to fixed SHAs, not a moving branch):

```bash
git submodule update --remote third_party/PointTransformerV3
git submodule update --remote third_party/object_condensation
# then commit the updated gitlink in the top-level repo
git add third_party/PointTransformerV3 third_party/object_condensation
git commit -m "Bump vendored submodules"
```
