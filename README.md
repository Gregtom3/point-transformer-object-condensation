# Point Transformer V3 + Object Condensation for Particle Reconstruction

## Project overview

This repository is a scaffold for a **multi-subdetector particle
reconstruction pipeline**. The physical setting is a collider event
that leaves hits in multiple subsystems ‚Äî silicon tracker, ECAL, HCAL,
RICH, etc. ‚Äî each of which records a different native representation of
the passing particle. The goal of reconstruction is to take the full
union of hits and recover the set of particles that produced them,
together with each particle's energy, momentum, and PID.

We treat each event as an unordered, variable-size point cloud whose
points live in 3D detector space. Hits from different subdetectors are
first embedded into a common feature width with per-subdetector linear
encoders and tagged with a learned subdetector-type token. The unified
token stream is fed through a **Point Transformer V3** (PTv3) backbone,
which provides a permutation-invariant, serialization-based attention
mechanism that scales well to O(10^4-10^5) hit events.

On top of the per-hit PTv3 features we attach three **Object
Condensation** (OC) heads: a condensation likelihood `beta`, a cluster
coordinate `x` in a learned low-dim space, and a payload head that
predicts the per-particle quantities (energy, momentum, PID). At
inference, hits above a beta threshold act as condensation points and
claim nearby hits in `x`-space, turning one-stage grid-free prediction
into a concrete particle list.

## Architecture at a glance

```
per-hit raw features
   ‚îÇ
   ‚îÇ  per-subdetector nn.Linear + learned subdetector-type embedding
   ‚ñº
unified (N, D) token stream
   ‚îÇ  + 3D coord, + offset (batch layout)
   ‚ñº
PointTransformerV3 backbone (serialized attention, enc/dec U-Net)
   ‚îÇ
   ‚ñº
per-hit features (N, D_out)
   ‚îÇ
   ‚îú‚îÄ‚îÄ beta head  (sigmoid)          ‚Üí condensation likelihood
   ‚îú‚îÄ‚îÄ x head                        ‚Üí cluster coordinates
   ‚îî‚îÄ‚îÄ payload head (E, p, PID)      ‚Üí per-hit per-particle prediction

Inference: Œ≤ > t_Œ≤ ‚üπ condensation point. Claim hits within ||x - x_c|| < t_d.
```

## Third-party components

Both backbone and loss code are vendored as **git submodules** under
`third_party/`:

| Submodule | Upstream | Pinned SHA | License |
|---|---|---|---|
| PointTransformerV3 | <https://github.com/Pointcept/PointTransformerV3> | `3229e9b7de1770c8ad17c316f8e349982de509f8` | MIT |
| object_condensation | <https://github.com/object-condensation/object_condensation> | `926827d7cf743a946a6738f8f590f6ad07644b6f` | MIT |

`PointTransformerV3` is imported directly from its source tree ‚Äî it is
not pip-installable. The wrapper at `src/models/backbone.py` prepends
its path to `sys.path`. `object_condensation` is a real Python package
and is installed in editable mode via
`pip install -e third_party/object_condensation[pytorch]` (see
`setup.sh`).

## Installation

The CUDA-linked wheels (`torch`, `torch-scatter`, `torch-geometric`,
`spconv`, `flash-attn`) must be installed in a particular order from
matching wheel indices. The easiest path is:

```bash
# clone with submodules
git clone --recursive https://github.com/gregtom3/point-transformer-object-condensation.git
cd point-transformer-object-condensation

# bootstrap (defaults to CUDA 12.1; override with CUDA=cu118 for 11.8)
CUDA=cu121 bash setup.sh
```

Manual pip path (CUDA 12.1 shown; swap `cu121` ‚Üí `cu118` to match your
driver):

```bash
pip install --extra-index-url https://download.pytorch.org/whl/cu121 "torch>=2.1"
pip install --find-links https://data.pyg.org/whl/torch-2.1.0+cu121.html torch-scatter torch-geometric
pip install spconv-cu121
pip install flash-attn --no-build-isolation      # optional, see below
pip install numpy scipy pandas pyyaml omegaconf hydra-core timm addict \
            wandb tensorboard pytest
pip install -e third_party/object_condensation[pytorch]
```

Conda path:

```bash
conda env create -f environment.yml
conda activate ptv3-oc-reco
```

### CUDA compatibility (read this if the install fails)

Every CUDA-linked dep in this stack ‚Äî `torch`, `torch-scatter`,
`torch-geometric`, `spconv`, `flash-attn` ‚Äî ships pre-built wheels only
for specific CUDA tags. **All of them must agree with the CUDA tag of
your `torch` build.** If torch is cu121, you need `spconv-cu121` and the
`torch-...+cu121.html` PyG wheel index; if they don't line up, pip will
either pick a wheel that segfaults at import or fail to find one at all.

Check what your torch was built against:

```bash
python -c "import torch; print(torch.__version__, torch.version.cuda)"
```

Supported combinations as of this writing:

| torch CUDA | spconv wheel    | PyG wheel index                                        |
|------------|-----------------|--------------------------------------------------------|
| cu121      | `spconv-cu121`  | `https://data.pyg.org/whl/torch-2.3.0+cu121.html`      |
| cu118      | `spconv-cu118`  | `https://data.pyg.org/whl/torch-2.1.0+cu118.html`      |

There are **no published `spconv-cu122+` / `spconv-cu130` wheels** yet.
If `torch.version.cuda` reports 12.2 or newer (e.g. a fresh `pip install
torch` pulling 2.5+/cu124 or 2.11+/cu130), downgrade torch to a tag that
spconv supports:

```bash
pip uninstall -y torch torchvision torchaudio
pip install --index-url https://download.pytorch.org/whl/cu121 \
    "torch==2.3.1" "torchvision==0.18.1" "torchaudio==2.3.1"

pip install --find-links https://data.pyg.org/whl/torch-2.3.0+cu121.html \
    torch-scatter torch-geometric
pip install spconv-cu121
```

If you want to skip spconv entirely (CPU-only dev, no reconstruction
runs), the OC loss and heads tests still pass ‚Äî only
`test_ptv3_forward_smoke` will be skipped.

### About flash-attn

PTv3 prefers [FlashAttention](https://github.com/Dao-AILab/flash-attention),
which requires CUDA ‚â• 11.6 and a reasonably modern Ampere+ GPU. If you
can't install it (CPU-only dev box, older GPU, build pain), **disable it
and shrink the patch sizes** in the model config:

```yaml
# configs/model/ptv3_base.yaml (or train/default.yaml)
backbone:
  enable_flash: false
  enc_patch_size: [128, 128, 128, 128, 128]
  dec_patch_size: [128, 128, 128, 128]
```

```bash
python scripts/train.py --config configs/train/default.yaml
```

## Shapes pseudo-dataset (end-to-end example)

A self-contained toy task: generate canvases of coloured geometric
shapes, treat every non-white pixel as a hit, and train the OC pipeline
to recover which hits belong to which shape plus each shape's class,
width, and height.

```bash
# 1. generate splits (train / val / test) with globally unique object ids
python data/generate_shapes.py --out data/shapes_v1 \
    --n-train 800 --n-val 100 --n-test 100 \
    --frame 128 128 --shapes-per-image 1 6 --shape-size 10 40 --seed 0

# 2. train (TB logs: runs/shapes, checkpoints: outputs/shapes)
python scripts/train.py --config configs/train/shapes.yaml

# 3. watch training in TensorBoard
tensorboard --logdir runs/shapes
# scalars: per-component OC loss, shape-CE, width/height MSE, grad norm
# images:  side-by-side truth vs. predicted cluster canvases
# embedding: OC cluster-coord space coloured by object_id
# histograms: beta distribution over training

# 4. evaluate a checkpoint (writes JSON summary)
python scripts/evaluate.py --config configs/train/shapes.yaml \
    --checkpoint outputs/shapes/epoch_009.pt --split test

# 5. export the heads to ONNX (backbone stays in PyTorch ‚Äî see caveat)
python scripts/export_onnx.py --config configs/train/shapes.yaml \
    --checkpoint outputs/shapes/epoch_009.pt --out outputs/shapes/heads.onnx
```

`scripts/train.py` writes `<log_dir>/architecture/architecture.md` on
startup: Mermaid flowchart, per-module parameter counts, and a
`torchinfo` layer table for the heads.

## Submodule maintenance

After a fresh clone without `--recursive`, populate the submodules
with:

```bash
git submodule update --init --recursive
```

To pull in newer upstream commits (explicitly ‚Äî the submodules are
pinned to fixed SHAs, not a moving branch):

```bash
git submodule update --remote third_party/PointTransformerV3
git submodule update --remote third_party/object_condensation
# then commit the updated gitlink in the top-level repo
git add third_party/PointTransformerV3 third_party/object_condensation
git commit -m "Bump vendored submodules"
```
